**Контекст кейса и бизнес-задача:**

Краудфандинговая платформа “Grass” (зарабатывает на комиссии за размещение проекта, а также на процентах от общего сбора проекта) захотела увеличить выручку за счёт развития текущего функционала, позволяющего определять вероятную успешность нового проекта, а также на какие сборы может рассчитывать автор. 

Основная задача проекта: 
1. Проанализировать данные (включая новые параметры и взаимосвязи между ними, которые не учтены в текущем алгоритме) с платформы, а также внешнее информационное окружение категорий  
2. Расширить число признаков для алгоритма, который, опираясь на категорию, тему (жанр), описание, количество дней сборов и требуемую сумму финансирования, наличие видео, будет выдавать автору проекта вероятность того, что его проект станет успешным (соберет нужную сумму) и укажет, в случае низкой вероятности успеха, на какие сборы может рассчитывать автор, за счет анализа внешней среды.

Обоснование задачи: 
Так как заказчик захотел развить функционал, который должен предсказывать, перед нами встала задача нахождения новых параметров для более детального и углубленного обучения модели. Для этого нам понадобится большой объем данных о проектах и их сборах, который мы будем собирать с краудфандинговой платформы и данные с внешних информационных ресурсов. Основная информация о проекте содержится в следующих признаках: категория, тема (подкатегория, жанр), описание, кол-во дней сборов, наличие видео, а также требуемая сумма финансирования, поэтому именно эти данные мы выбрали основными для анализа и дальнейшего дообучения.

 ## Структура проекта

**branch eda/**
- **.gitignore** - список игнорируемых гитом файлов
- **calculator.py** - класс, генерирующий новые признаки
- **README.md** - описание проекта
- **Init_gp_file.ipynb** - изначальный файл группового проекта
- **requirements.txt** - список используемых библиотек Python

- **data/** - папка с данными, полученными в ходе работы
  - Description_Kickstarter.xlsx - описание признаков с Kickstarter
  - comicsq.csv - датасет (получен с eBay) по комиксам
  - game_categories.py - все жанры игр
  - google_play_games.csv - датасет, полученный с Google Play
  - google_play_games_id.csv - id игр, полученные с Google Play
  - kickstarter.csv - датасет, полученный с Kickstarter
  - kickstarter_preparation.csv - датасет, полученный с Kickstarter после обработки
  - movies_final.xlsx - датасет, полученный с Кинопоиска
  - music_top_LastFM.xlsx - датасет, полученный с Last FM
  - steam_games.csv - датасет, полученный с Steam
  - steam_games_id.csv - id игр, полученные с Steam

- **eda/** - папка с notebooks для анализа данных
  - EDA_comics.ipynb - анализ данных с eBay
  - games_analisys.ipynb - анализ данных с Google Play и Steam
  - kickstarter_common.ipynb - анализ данных с Kickstarter
  - kickstarter_preparation.ipynb - предобработка данных с Kickstarter
  - kinopoisk.ipynb - анализ данных с Кинопоиска
  - music_analysis.ipynb - анализ данных с Last FM

- **logs/** - папка с логами
  - google_play.log - логи с Google Play
  - kickstarter.log - логи с Kickstarter
  - calculator.log - логи с запуска калькулятора
  - steam_games.log - логи с Steam

- **scraping/** - папка с notebooks для получения данных
  - Api2_kinopoisk.ipynb - получение данных КП с помощью API-key
  - comics_API.ipynb - получение данных eBay с помощью API-key
  - google_play_games.py - получение данных GP с помощью API-key
  - google_play_ids.py - получение id игр GP с помощью API-key
  - music_top_from_LastFM.ipynb - получение данных LFM с помощью API-key
  - parse.py - парсинг данных с Kickstarter
  - run_scraper.bash - скрипт для запуска парсинга данных с Kickstarter
  - steam_games.py - получение данных Steam с помощью API-key
  - steam_id.py - получение id игр Steam с помощью API-key
